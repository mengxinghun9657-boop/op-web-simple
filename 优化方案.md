# op-web-simple 系统优化方案

## 一、Elasticsearch 替代数据库查询场景

### 1.1 当前 MySQL 查询痛点

| 场景 | 当前实现 | 问题 |
|------|---------|------|
| **CMDB 资源搜索** | 多表关联 + LIKE 模糊查询 | 性能差、代码复杂 |
| **告警记录筛选** | 多条件组合 + 复合索引 | 索引爆炸、查询慢 |
| **故障手册匹配** | 精确匹配 | 无法全文检索 |
| **审计日志** | 未实现 | 需要全文检索能力 |

### 1.2 适合迁移到 ES 的场景

#### 🔴 高优先级

**1. CMDB 资源搜索** ([cmdb.py](backend/app/api/v1/cmdb.py))
- 当前：多表关联查询（服务器 ↔ 实例 ↔ BCE数据）
- 问题：模糊搜索使用 `LIKE '%keyword%'`，批量关键词需要循环处理
- ES 优势：统一全文检索、跨字段搜索、毫秒级响应

```python
# 当前 MySQL 实现
query.filter(
    or_(
        IaasServer.bns_hostname.contains(kw),
        IaasServer.rms_sn.contains(kw),
        IaasServer.rms_ip_in1.contains(kw),
    )
)

# ES 实现
{
  "query": {
    "multi_match": {
      "query": "keyword",
      "fields": ["hostname^3", "sn", "ip", "rack_info", "uuid"]
    }
  }
}
```

**2. 告警记录多维筛选** ([alerts.py](backend/app/api/v1/endpoints/alerts.py))
- 当前：多条件组合查询（类型、严重程度、组件、IP、集群ID、时间范围）
- 问题：IP 和集群 ID 使用 `LIKE` 模糊匹配，需要多个复合索引
- ES 优势：倒排索引快速过滤、聚合统计毫秒级

#### 🟡 中优先级

**3. 故障手册全文检索** ([manual_matcher.py](backend/app/services/alert/manual_matcher.py))
- 当前：基于告警类型精确匹配
- ES 增强：全文检索故障现象、语义相似度匹配、智能推荐

**4. 审计日志/操作记录**
- 当前：缺失
- ES 方案：用户操作轨迹追踪、多维度筛选、操作内容全文检索

#### 🟢 低优先级

**5. 监控指标元数据搜索** ([pfs.py](backend/app/api/v1/pfs.py))
- 场景：PFS 指标目录查询、指标名称/描述/标签搜索
- 优先级：数据量小，MySQL 可满足

### 1.3 架构建议

```
┌─────────────────────────────────────────────────────────┐
│                        前端界面                          │
└─────────────────────────────────────────────────────────┘
                           │
    ┌──────────────────────┼──────────────────────┐
    ▼                      ▼                      ▼
┌─────────┐         ┌──────────┐          ┌──────────┐
│ MySQL   │         │   ES     │          │ Prometheus│
│ (事务性) │         │ (搜索/分析)│          │ (时序数据) │
├─────────┤         ├──────────┤          ├──────────┤
│告警记录  │         │日志全文   │          │PFS指标    │
│CMDB基础  │◄──────►│CMDB搜索   │          │集群监控   │
│用户信息  │  同步   │告警搜索   │          │          │

└─────────┘         └──────────┘          └──────────┘
```

**关键决策**：ES 作为 **搜索增强层**，而非完全替代 MySQL。MySQL 保留事务性操作，ES 负责复杂搜索和分析。

## 二、长时间任务解耦方案

### 2.1 当前问题分析

#### 长时间任务现状

| 任务类型 | 执行时间 | 当前实现 | 问题 |
|---------|---------|---------|------|
| **资源分析** ([resource_service.py](backend/app/services/resource_service.py)) | 30s-5min | BackgroundTasks | 阻塞主进程、内存占用大 |
| **EIP分析** ([eip_service.py](backend/app/services/eip_service.py)) | 10s-2min | BackgroundTasks | BCM API调用慢 |
| **运营数据分析** ([operational_service.py](backend/app/services/operational_service.py)) | 20s-3min | BackgroundTasks | AI解读耗时 |
| **PFS数据导出** | 1min-10min | BackgroundTasks | 大数据量查询 |
| **Prometheus批量采集** | 30s-5min | BackgroundTasks | 多集群轮询 |
| **CMDB同步** | 1min-10min | 定时任务 | 大数据量同步 |

#### 当前架构问题

```

│  ┌─────────┐ ┌─────────┐ ┌──────────┐  │

│       └───────────┴───────────┘         │
│              共享进程/内存               │

              ❌ 问题：
              - 后台任务阻塞API响应
```

#### 3. Worker 代码示例

```python
# worker/main.py
import os
import asyncio
from celery import Celery
from loguru import logger

# 创建 Celery 应用
celery_app = Celery('tasks', broker=os.getenv('REDIS_URL', 'redis://redis:6379/0'))

# 根据环境变量注册不同的任务队列
worker_queues = os.getenv('WORKER_QUEUES', 'analytics').split(',')

# ============ 分析类任务 ============
if 'analytics' in worker_queues:
    @celery_app.task(name='analytics.resource', bind=True)
    def analyze_resource(self, cluster_ids, excel_file):
        """资源分析任务"""
        from app.services.resource_service import analyze_resource_data
        
        logger.info(f"[Resource Worker] 开始分析任务: {self.request.id}")
        self.update_state(state='PROGRESS', meta={'progress': 10})
        
        result = asyncio.run(analyze_resource_data(
            task_id=self.request.id,
            cluster_ids=cluster_ids,
            excel_file_path=excel_file
        ))
        
        self.update_state(state='PROGRESS', meta={'progress': 90})
        logger.info(f"[Resource Worker] 分析完成: {self.request.id}")
        return result

    @celery_app.task(name='analytics.eip', bind=True)
    def analyze_eip(self, eip_ids, hours):
        """EIP分析任务"""
        from app.services.eip_service import eip_service
        
        logger.info(f"[EIP Worker] 开始分析: {self.request.id}")
        result = asyncio.run(eip_service.analyze_eip(eip_ids, hours))
        logger.info(f"[EIP Worker] 分析完成: {self.request.id}")
        return result

    @celery_app.task(name='analytics.operational', bind=True)
    def analyze_operational(self, file_path, task_id):
        """运营数据分析任务"""
        from app.services.operational_service import analyze_operational_file
        
        logger.info(f"[Operational Worker] 开始分析: {task_id}")
        result = asyncio.run(analyze_operational_file(file_path, task_id))
        logger.info(f"[Operational Worker] 分析完成: {task_id}")
        return result

# ============ 采集类任务 ============
if 'collector' in worker_queues:
    @celery_app.task(name='collector.cmdb_sync', bind=True)
    def sync_cmdb(self, azone):
        """CMDB同步任务"""
        from app.services.cmdb_sync_service import CMDBSyncService
        
        logger.info(f"[Collector Worker] 开始CMDB同步: {azone}")
        return {"success": True, "synced": 100}

    @celery_app.task(name='collector.prometheus', bind=True)
    def collect_prometheus(self, cluster_ids):
        """Prometheus数据采集"""
        from app.services.prometheus_service import PrometheusService
        
        logger.info(f"[Collector Worker] 开始采集: {cluster_ids}")
        service = PrometheusService()
        return service.get_multiple_clusters_metrics(cluster_ids)

# ============ 告警类任务 ============
if 'alert' in worker_queues:
    @celery_app.task(name='alert.process_file', bind=True)
    def process_alert_file(self, file_path):
        """处理告警文件"""
        from app.services.alert.file_watcher import FileWatcherService
        
        logger.info(f"[Alert Worker] 处理告警文件: {file_path}")
        watcher = FileWatcherService()
        return watcher.process_file(file_path)

    @celery_app.task(name='alert.diagnose', bind=True)
    def diagnose_alert(self, alert_data):
        """告警诊断任务"""
        from app.services.alert.alert_processor import AlertProcessor
        
        logger.info(f"[Alert Worker] 诊断告警: {alert_data.get('id')}")
        processor = AlertProcessor()
        return asyncio.run(processor.process_single_alert(alert_data))


if __name__ == '__main__':
    queue_names = ','.join(worker_queues)
    logger.info(f"启动Worker，监听队列: {queue_names}")
    
    celery_app.start([
        'worker',
        '--loglevel=info',
        '--queues', queue_names,
        '--concurrency=4',
        '--prefetch-multiplier=1',
    ])
```

#### 4. 主服务调用方式

```python
# backend/app/api/v1/resource.py (修改后)
from fastapi import APIRouter
from celery import Celery

router = APIRouter()
celery_app = Celery('tasks', broker=os.getenv('REDIS_URL'))

@router.post("/resource/analyze")
async def start_resource_analysis(request: ResourceRequest):
    """
    提交资源分析任务
    不再使用 BackgroundTasks，而是提交到 Celery 队列
    """
    task = celery_app.send_task(
        'analytics.resource',
        args=[request.cluster_ids, request.excel_file_name],
        queue='analytics'
    )
    
    save_task_to_db(task.id, TaskType.RESOURCE, TaskStatus.PENDING)
    
    return {
        "task_id": task.id,
        "status": "pending",
        "message": "任务已提交，正在排队处理"
    }

@router.get("/tasks/{task_id}/status")
async def get_task_status(task_id: str):
    """查询任务状态"""
    result = celery_app.AsyncResult(task_id)
    
    return {
        "task_id": task_id,
        "status": result.status,
        "result": result.result if result.ready() else None
    }
```

### 2.4 关键优势

## 五、容器日志管理优化方案

### 5.1 当前日志管理现状

#### 当前架构分析

| 组件 | 当前实现 | 存在问题 |
|------|---------|---------|
| **应用日志** ([logger.py](backend/app/core/logger.py)) | loguru 写入文件 | 容器内日志文件，重启后丢失 |

#### 添加日志追踪中间件

```python
# backend/app/middleware/logging.py
import time
import uuid
from fastapi import Request
from loguru import logger

async def logging_middleware(request: Request, call_next):
    """请求日志追踪中间件"""
    request_id = str(uuid.uuid4())[:8]
    request.state.request_id = request_id
    
    start_time = time.time()
    
    # 记录请求开始
    logger.info({
        "event": "request_start",
        "request_id": request_id,
        "method": request.method,
        "path": request.url.path,
        "client_ip": request.client.host if request.client else None,
        "user_agent": request.headers.get("user-agent"),
    })
    
    response = await call_next(request)
    
    # 记录请求结束
    duration = time.time() - start_time
    logger.info({
        "event": "request_end",
        "request_id": request_id,
        "method": request.method,
        "path": request.url.path,
        "status_code": response.status_code,
        "duration_ms": round(duration * 1000, 2),
    })
    
    # 添加请求ID到响应头
    response.headers["X-Request-ID"] = request_id
    return response
```

### 8.1 缓存管理 → **引入 cachetools + redis-py 高级封装**

#### 当前问题
```python
# backend/app/core/redis_client.py - 当前实现
class RedisClient:
    """Redis客户端封装"""
    
    def set_cache(self, key: str, value: Any, expire: int = 300):
        """设置缓存"""
        val = json.dumps(value, ensure_ascii=False)
        self.client.setex(key, expire, val)
    
    def get_cache(self, key: str) -> Optional[Any]:
        """获取缓存"""

```

**问题**：
- 没有缓存策略（LRU/LFU）
- 没有多级缓存（本地 + 远程）
- 没有缓存穿透/击穿/雪崩防护
- 每个调用点都要手动处理序列化

#### 推荐组件方案

```python
# 方案一：cachetools（本地缓存）+ redis（分布式缓存）
from cachetools import TTLCache, LRUCache
from functools import wraps
import hashlib
import json

class MultiLevelCache:
    """多级缓存组件"""
    
    def __init__(self, redis_client):
        # L1: 本地内存缓存（10MB，5分钟过期）
        self.local_cache = TTLCache(maxsize=1000, ttl=300)
        # L2: Redis 分布式缓存
        self.redis = redis_client
    
    def cache_decorator(self, ttl=300, key_prefix="cache"):
        """缓存装饰器"""
        def decorator(func):
            @wraps(func)
            def wrapper(*args, **kwargs):
                # 生成缓存key
                cache_key = f"{key_prefix}:{func.__name__}:{self._generate_key(args, kwargs)}"
                
                # L1: 尝试本地缓存
                if cache_key in self.local_cache:
                    return self.local_cache[cache_key]
                
                # L2: 尝试Redis缓存
                value = self.redis.get(cache_key)
                if value:
                    self.local_cache[cache_key] = value
                    return value
                
                # 执行函数
                result = func(*args, **kwargs)
                
                # 写入两级缓存
                self.local_cache[cache_key] = result
                self.redis.setex(cache_key, ttl, json.dumps(result))
                
                return result
            return wrapper
        return decorator
    
    def _generate_key(self, args, kwargs):
        """生成缓存key"""
        key_data = json.dumps({"args": args, "kwargs": kwargs}, sort_keys=True, default=str)
        return hashlib.md5(key_data.encode()).hexdigest()[:16]

# 使用示例

    """获取服务器详情 - 自动缓存"""
    return db.query(IaasServer).filter(...).first()
```

```python
# 方案：tenacity（重试+熔断）+ slowapi（限流）
from tenacity import (
    retry, stop_after_attempt, wait_exponential, 
    retry_if_exception_type, CircuitBreaker
)
from slowapi import Limiter
from functools import wraps
import time

class ResilienceDecorator:
    """弹性装饰器组件（限流+重试+熔断）"""
    
    def __init__(self):
        self.breakers = {}
    
    def circuit_breaker(self, failure_threshold=5, recovery_timeout=60):
        """熔断器装饰器"""
        def decorator(func):
            func_name = func.__name__
            if func_name not in self.breakers:
                self.breakers[func_name] = CircuitBreaker(
                    failure_threshold=failure_threshold,
                    recovery_timeout=recovery_timeout,
                    expected_exception=Exception
                )
            
            @wraps(func)
            def wrapper(*args, **kwargs):
                breaker = self.breakers[func_name]
                if breaker.current_state == "OPEN":
                    raise Exception(f"服务熔断: {func_name}")
                
                try:
                    result = func(*args, **kwargs)
                    breaker.record_success()
                    return result
                except Exception as e:
                    breaker.record_failure()
                    raise e
            
            return wrapper
        return decorator
    
    def retry_with_backoff(self, max_attempts=3, min_wait=1, max_wait=10):
        """指数退避重试"""
        return retry(
            stop=stop_after_attempt(max_attempts),
            wait=wait_exponential(multiplier=1, min=min_wait, max=max_wait),
            retry=retry_if_exception_type((requests.RequestException, TimeoutError))
        )

# 使用示例
resilience = ResilienceDecorator()

@resilience.circuit_breaker(failure_threshold=3, recovery_timeout=30)
@resilience.retry_with_backoff(max_attempts=3)
def call_prometheus_api(query: str):
    """调用 Prometheus API - 带熔断和重试"""
    response = requests.get(url, params={"query": query}, timeout=10)
    return response.json()
```

| **pybreaker** | 熔断器 | 专业的熔断实现 |

```python
# 方案：httpx（支持异步 + 同步）
import httpx
from typing import Optional

class HTTPClient:
    """HTTP 客户端组件"""
    
    def __init__(self):
        # 异步客户端（用于 FastAPI）
        self.async_client = httpx.AsyncClient(
            timeout=httpx.Timeout(30.0, connect=5.0),
            limits=httpx.Limits(max_keepalive_connections=20, max_connections=100),
            http2=True,  # 启用 HTTP/2
        )
        
        # 同步客户端（用于后台任务）
        self.sync_client = httpx.Client(
            timeout=httpx.Timeout(30.0, connect=5.0),
            limits=httpx.Limits(max_keepalive_connections=20, max_connections=100),
        )
    
    async def get_async(self, url: str, **kwargs):
        """异步 GET 请求"""
        response = await self.async_client.get(url, **kwargs)
        response.raise_for_status()
        return response
    
    def get_sync(self, url: str, **kwargs):
        """同步 GET 请求"""
        response = self.sync_client.get(url, **kwargs)
        response.raise_for_status()
        return response

# 使用示例
```

```python
from prometheus_client import Counter, Histogram, Gauge, generate_latest, CONTENT_TYPE_LATEST
from opentelemetry import trace
from opentelemetry.exporter.otlp.proto.grpc.trace_exporter import OTLPSpanExporter
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from functools import wraps
import time

# ========== Prometheus 指标 ==========
class MetricsCollector:
    """指标收集组件"""
    
    def __init__(self):
        # 请求计数器
        self.request_count = Counter(
            'http_requests_total',
            'Total HTTP requests',
            ['method', 'endpoint', 'status_code']
        )
        
        # 请求耗时直方图
        self.request_duration = Histogram(
            'http_request_duration_seconds',
            'HTTP request duration',
            ['method', 'endpoint'],
            buckets=[0.1, 0.5, 1.0, 2.0, 5.0, 10.0]
        )
        
        # 活跃任务数
        self.active_tasks = Gauge(
            'active_tasks',
            'Number of active tasks',
            ['task_type']
        )
    
    def track_request(self, method: str, endpoint: str, status_code: int, duration: float):
        """记录请求指标"""
        self.request_count.labels(method=method, endpoint=endpoint, status_code=status_code).inc()
        self.request_duration.labels(method=method, endpoint=endpoint).observe(duration)

# ========== OpenTelemetry 链路追踪 ==========
class TracingConfig:
    """链路追踪配置"""
    
    def __init__(self, service_name: str = "op-web-backend"):
        # 配置 tracer
        provider = TracerProvider()
        
        # OTLP 导出器（发送到 Jaeger/Tempo）
        otlp_exporter = OTLPSpanExporter(endpoint="http://jaeger:4317", insecure=True)
        provider.add_span_processor(BatchSpanProcessor(otlp_exporter))
        
        trace.set_tracer_provider(provider)
        self.tracer = trace.get_tracer(service_name)
    
    def trace_decorator(self, span_name: str = None):
        """追踪装饰器"""
        def decorator(func):
            @wraps(func)
            def wrapper(*args, **kwargs):
                name = span_name or func.__name__
                with self.tracer.start_as_current_span(name) as span:
                    span.set_attribute("function.name", func.__name__)
                    span.set_attribute("function.args", str(args))
                    result = func(*args, **kwargs)
                    span.set_attribute("function.result", "success")
                    return result
            return wrapper
        return decorator

        duration=duration
    )
    
    return response
```
